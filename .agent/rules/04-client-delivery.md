# Client Delivery

Checklist before closing and handing off a project:

### Code & Repo
- [ ] All code merged to `main`, no stale branches
- [ ] README.md with setup instructions
- [ ] `.env.example` documented

### Documentation
- [ ] `docs/` finalized (discovery, requirements, architecture, plan)
- [ ] Automations documented in `automations/` (if applicable)

### Infrastructure
- [ ] Production running, domain + SSL configured
- [ ] DB backups configured
- [ ] Monitoring set up (if applicable)

### Access
- [ ] Client has access to: repo, hosting, DB, external services
- [ ] Credentials shared via secure channel
- [ ] Antigravity access removed or scoped down

### Handoff
- [ ] Walkthrough session done or recorded
- [ ] Known limitations documented
